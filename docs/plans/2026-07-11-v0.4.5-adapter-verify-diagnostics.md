# v0.4.5 Adapter Verify Diagnostics Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Make `verify --ide <id>` and `doctor` validate the generated runtime files for Codex, Cursor, Claude Code, and Trae instead of only checking Codex's minimal runtime paths.

**Architecture:** Keep adapter runtime definitions in the CLI verifier as a small explicit map keyed by supported IDE. Reuse existing `missing-ide-runtime` issue code for missing generated adapter files, and make doctor remediation derive the correct `sync --ide <id>` command from the issue path.

**Tech Stack:** TypeScript CLI, Vitest, existing `verifyProject`, existing `diagnoseProject`, adapter runtime outputs from v0.4.1 to v0.4.4.

## Current Baseline

`apps/cli/src/lib/verify.ts` currently checks only Codex runtime:

```ts
async function verifyIdeRuntime(projectRoot: string, ide: string, issues: VerificationIssue[]): Promise<void> {
  if (ide === "codex") {
    if (!(await fs.pathExists(path.join(projectRoot, ".codex", "ai-video-workflow", "WORKFLOW_OVERVIEW.md")))) {
      pushIssue(issues, {
        code: "missing-ide-runtime",
        message: "Missing Codex runtime mirror",
        path: ".codex/ai-video-workflow"
      });
    }
    if (!(await fs.pathExists(path.join(projectRoot, ".codex", "skills")))) {
      pushIssue(issues, {
        code: "missing-ide-runtime",
        message: "Missing Codex runtime skills",
        path: ".codex/skills"
      });
    }
  }
}
```

`apps/cli/src/lib/doctor.ts` always suggests:

```text
Run `ai-video-workflow sync --project <path> --ide codex` to restore the IDE runtime files.
```

This is now stale because Cursor, Claude Code, and Trae have real runtime mirrors.

## Design Decisions

- Keep the issue code `missing-ide-runtime` to avoid expanding issue taxonomy prematurely.
- Include IDE name in the issue message, for example `Missing Cursor runtime file: .cursor/rules/ai-video-workflow.mdc`.
- Store path as the exact missing relative path, not just the parent directory, so doctor can infer the IDE from path.
- Add a small `ideFromRuntimePath` helper in doctor:
  - `.codex/` -> `codex`
  - `.cursor/` -> `cursor`
  - `.claude/` or `CLAUDE.md` -> `claude-code`
  - `.trae/` -> `trae`
  - fallback -> `codex`
- Keep `AGENTS.md` ambiguous in doctor fallback unless paired with a Trae-specific message. Since Trae is the only adapter whose verifier requires `AGENTS.md`, use the message/path combination to suggest Trae.
- Do not read `project.config.yaml` IDE to infer doctor command because doctor receives only issues, not project config.

**Step 3: Add helper in doctor**

Add:

```ts
function ideForRuntimeIssue(issue: VerificationIssue): string {
  const marker = `${issue.path ?? ""} ${issue.message}`;
  if (marker.includes(".cursor/")) {
    return "cursor";
  }
  if (marker.includes(".claude/") || marker.includes("CLAUDE.md") || marker.includes("Claude Code")) {
    return "claude-code";
  }
  if (marker.includes(".trae/") || marker.includes("Trae")) {
    return "trae";
  }
  return "codex";
}
```
