# v0.4.4 Trae Adapter Hardening Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Harden the Trae adapter against the agent adapter contract so Trae receives native `.trae/` skills, specs, documents, and boundary rules without relying on Claude-specific compatibility files.

**Architecture:** Keep `packs/official-ai-video/` and project Step 1 to Step 6 files as the only workflow truth. Treat `.trae/` as Trae's generated runtime mirror, with `.trae/skills/` as native skill bundles, `.trae/specs/ai-video-workflow/` as workflow specs, and `.trae/documents/ai-video-workflow/` as the human/agent-readable mirror.

**Tech Stack:** TypeScript CLI, Vitest, Markdown docs, YAML sync rules, Trae `.trae/skills/`, `.trae/specs/`, `.trae/documents/`, and `.trae/rules/` runtime files.

---

## Preconditions

- Cursor v0.4.2 and Claude Code v0.4.3 adapter hardening must already be committed.
- Worktree must be clean before starting implementation.
- Do not mix Trae work with adapter diagnostics, MCP, Obsidian, or LibTV changes.
- Keep all links relative.
- Do not make Trae files a second workflow source.

## Scope

This milestone upgrades Trae from a minimal `.trae/skills` copy into a complete adapter-contract runtime mirror.

In scope:

- Generate `.trae/skills/` from pack skill bundles.
- Generate `.trae/specs/ai-video-workflow/` from pack workflow docs.
- Generate `.trae/documents/ai-video-workflow/` with `WORKFLOW_OVERVIEW.md`, longform skills, skill bundles, templates, and indexes.
- Generate `.trae/rules/ai-video-workflow.md` as a short Trae-native runtime rule entry.
- Generate `AGENTS.md` as a generic compatibility entry only if missing.
- Stop generating `CLAUDE.md` from `sync --ide trae`; Claude-specific entrypoints belong to `sync --ide claude-code`.
- Add tests proving required Trae files exist and generated text contains no absolute local links.
- Update sync rules and Trae docs to match actual output.
- Add a Trae adapter contract fixture.

Out of scope:

- No Trae-specific rewrite of Step 1 to Step 6 rules.
- No automatic edits to source Step files.
- No `.obsidian/` writes.
- No generated Obsidian projection changes.
- No MCP server or LibTV execution work.
- No use of `.agents/skills` as a default Trae target.

## Current Baseline

Current implementation in `apps/cli/src/lib/sync.ts`:

```ts
async function syncTrae(projectRoot: string, packRoot: string): Promise<void> {
  await copyDirectory(path.join(packRoot, "skills"), path.join(projectRoot, ".trae", "skills"));
  await fs.ensureDir(path.join(projectRoot, ".trae", "rules"));
  await writeFileIfMissing(path.join(projectRoot, "AGENTS.md"), "# Trae Compatibility Entry\n");
  await writeFileIfMissing(path.join(projectRoot, "CLAUDE.md"), "# Trae Compatibility Entry\n");
}
```

Current docs:

- `packs/official-ai-video/ide-integrations/trae/README.md` recommends `.trae/skills/`, `.trae/specs/ai-video-workflow/`, and `.trae/documents/ai-video-workflow/`.
- `docs/en/ide-integrations/trae.md` and `docs/zh/ide-integrations/trae.md` only document `.trae/skills`.
- `packs/official-ai-video/checks/sync-rules.yaml` only records `skillsDir: ".trae/skills"`.

## Design Decisions

- Use `.trae/` only for Trae-native runtime output.
- Keep `AGENTS.md` as a generic compatibility entry because Trae may read generic agent guidance, but keep it short and non-authoritative.
- Remove `CLAUDE.md` generation from `sync --ide trae` because it is misleading and belongs to Claude Code, not Trae.
- Copy `packs/official-ai-video/workflow/` into `.trae/specs/ai-video-workflow/` because the pack README identifies specs as the workflow-spec target.
- Copy longform skills, skill bundles, templates, and indexes into `.trae/documents/ai-video-workflow/` for readable mirror parity.
- Use `writeFileIfMissing` for `.trae/rules/ai-video-workflow.md` and `AGENTS.md` to preserve user-authored project guidance.
- Use `copyDirectory` for adapter-owned mirror directories.
