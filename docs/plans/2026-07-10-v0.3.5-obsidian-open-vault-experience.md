# v0.3.5 Obsidian Open Vault Experience Implementation Plan

## Goal

Make the generated Obsidian vault easier to use immediately after opening. The first path should be clear: open Project Home, inspect shots, use Review Map for spatial navigation, copy Agent Handoff context into an agent chat, then rerun verification after source Step edits.

## Architecture Decisions

- Keep Obsidian as a generated viewing and handoff layer, not a workflow source.
- Keep `export-obsidian` and `verify-obsidian` interfaces unchanged.
- Keep `.obsidian/` writes opt-in through `--include-obsidian-ui`.
- Do not add plugins, reverse sync, agent execution, or LibTV work.
- Preserve relative links and vault-relative Canvas/Workspace/Bookmark paths.

## Atomic Steps

1. Improve generated vault entry copy in `dashboard.ts`.
   - Update generated vault `README.md` to point first to `00_Project_Home`, then `04_Agent_Handoff`, `02_Shot_Index`, and `Canvas/Review Map.canvas`.
   - Add an `Open Vault Workflow` section to `00_Project_Home.md`.
   - The section must describe four actions: inspect project, inspect a shot, hand off to agent, verify after edits.
   - Keep source-of-truth boundary text visible on the home page.

2. Refine dashboard navigation.
   - Add a short `Start Here` style entry list to Project Home without removing existing Bases and Canvas embeds.
   - Make `04_Agent_Handoff.md` visible near the top of Project Home.
   - Keep Review Dashboard and Production Board focused on review and readiness, but link back to the open-vault flow where useful.

3. Update opt-in UI suggestions in `ui-config.ts`.
   - Add Bookmarks for `00_Project_Home.md`, `04_Agent_Handoff.md`, `02_Shot_Index.md`, `Canvas/Review Map.canvas`, `Canvas/Shot Pipeline.canvas`, and `Notes/README.md`.
   - Update `workspace.json` so the main split opens Project Home and Agent Handoff side by side.
   - Add `Canvas/Review Map.canvas`, `02_Shot_Index.md`, and `03_Production_Board.md` to `lastOpenFiles`.
   - Continue writing direct `.obsidian/*.json` only when `--include-obsidian-ui` is enabled, and continue writing suggested copies under `.obsidian/ai-video-workflow-suggested/`.

4. Add tests.
   - Export tests assert Project Home contains `Open Vault Workflow` and the four intended actions.
   - UI config tests assert bookmarks include Agent Handoff, Shot Index, Review Map, and Shot Pipeline.
   - UI config tests assert workspace opens Project Home and Agent Handoff and uses vault-relative file paths.
   - Existing default-no-`.obsidian/` and no-overwrite tests must continue to pass.

5. Update docs.
   - README and quickstart explain the exact open-vault path.
   - Contributor docs explain that `--include-obsidian-ui` can pre-bookmark the main route but remains optional and non-authoritative.

6. Run validation.
