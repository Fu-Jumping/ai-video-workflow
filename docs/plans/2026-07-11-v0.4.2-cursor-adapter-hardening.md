# v0.4.2 Cursor Adapter Hardening Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Harden the Cursor adapter against the agent adapter contract without introducing Cursor-specific workflow rules.

**Architecture:** Keep `packs/official-ai-video/` and project Step files as the source of truth. Treat `.cursor/` as a runtime mirror and native rules surface generated from the official pack, with tests proving required output locations and relative-link safety.

**Tech Stack:** TypeScript CLI, Vitest, Markdown docs, Cursor `.cursor/` runtime files, existing `sync --ide cursor` command.

---

## Scope

This milestone is the first platform rollout after the agent adapter contract. It only hardens Cursor behavior and docs. It does not change the core workflow, Obsidian projection, LibTV, MCP, or generation-platform calls.

## Current Baseline

- `syncProject` currently copies `packs/official-ai-video/skills` to `.cursor/skills`.
- `docs/en/ide-integrations/cursor.md` and `docs/zh/ide-integrations/cursor.md` describe the adapter contract at a high level.
- `packs/official-ai-video/ide-integrations/cursor/README.md` expects `.cursor/rules/` and `.cursor/ai-video-workflow/` as richer Cursor-native targets.
- `packs/official-ai-video/checks/sync-rules.yaml` currently records only `.cursor/skills`.

## Design Decisions

- Add Cursor runtime mirror output in the CLI rather than leaving docs ahead of behavior.
- Keep `.cursor/skills/` as the minimal skill bundle output.
- Add `.cursor/ai-video-workflow/` mirror output for overview, longform skills, skill bundles, templates, and indexes.
- Add a small `.cursor/rules/ai-video-workflow.mdc` entrypoint that points to the runtime mirror and repeats only boundary rules.
- Do not create per-step Cursor rules in this slice.
- Do not copy generated Obsidian files into Cursor runtime.
- Do not write absolute paths.
