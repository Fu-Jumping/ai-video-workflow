# Post v0.3 Build Roadmap Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Consolidate the finished Obsidian v0.3 line into a releasable baseline, then define a platform-neutral agent adapter contract before adding platform-specific integrations.

**Architecture:** Keep `packs/official-ai-video/` and project Step 1 to Step 6 files as the only workflow truth. Treat Obsidian, Codex, Cursor, Claude Code, Trae, MCP, and future LibTV support as adapters with explicit inputs, outputs, sync direction, failure behavior, and verification gates.

**Tech Stack:** TypeScript CLI, Markdown docs, VitePress, Vitest, generated Obsidian projection files, JSON/YAML schemas where contracts need machine-readable checks.

---

## Current Baseline

The current repository has completed the Obsidian v0.3 line through v0.3.6:

- v0.3: Obsidian vault projection.
- v0.3.1: safe incremental export and manifest/hash protection.
- v0.3.2: project overview and review dashboard experience.
- v0.3.3: single-shot immersive review pages and shot review canvases.
- v0.3.4: agent handoff pages and copy-ready prompts.
- v0.3.5: open-vault workflow and optional UI suggestions.
- v0.3.6: real-vault QA, optional UI validation, and `example:obsidian:ui`.

The next work should not keep expanding Obsidian by default. The next stable path is:

1. Consolidate and publish the v0.3 Obsidian baseline.
2. Define a platform-neutral agent adapter contract.
3. Implement narrow platform adapters against that contract.

## Non-Goals

- Do not introduce LibTV execution work in the immediate next slice.
- Do not develop an Obsidian plugin.
- Do not introduce reverse sync from Obsidian to Step files.
- Do not make any platform adapter a second source of truth.
- Do not automate image/video generation platforms.
- Do not add platform-specific behavior before the adapter contract is documented and verified.

## Milestone B: v0.4 Agent Adapter Contract

### Purpose

Define one platform-neutral contract that lets Codex, Cursor, Claude Code, Trae, MCP, and future adapters consume the workflow consistently.

The goal is not to automate every platform. The goal is to prevent each platform from inventing its own project interpretation.

### Contract Principles

- Step files remain the source of truth.
- Adapter output is generated or synchronized view/runtime state.
- Agent handoff should say which source files to read and edit.
- Adapters must expose verification commands.
- Platform-specific config must not leak into core workflow rules.
- All documentation links remain relative.

### Proposed Contract Shape

Create a small machine-readable contract under `schemas/agent-adapter-contract.schema.json` and docs under contributors.

The contract should describe:

- `adapterId`: `codex`, `cursor`, `claude-code`, `trae`, `mcp`, or future id.
- `displayName`.
- `inputs`: project root, pack root, Step files, runtime mirror files.
- `outputs`: generated files or runtime config locations.
- `syncDirection`: `one-way-project-to-adapter`, `runtime-mirror`, or `read-only-context`.
- `sourceOfTruth`: expected to be `project-step-files`.
- `handoffSurfaces`: prompt files, generated Obsidian pages, IDE rules, MCP resources.
- `verificationCommands`.
- `forbiddenWrites`: generated projection files, `.obsidian` user config by default, platform cache directories, absolute links.
- `failureBehavior`: skip, report, preserve user files, rollback guidance.
