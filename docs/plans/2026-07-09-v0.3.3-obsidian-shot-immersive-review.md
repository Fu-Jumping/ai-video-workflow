# v0.3.3 Obsidian Shot Immersive Review Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Build an Obsidian single-shot immersive review surface so each shot page becomes a focused review hub for storyboard, image prompt, video prompt, execution readiness, canvas route, and user notes.

**Architecture:** Keep the project Step files as the only workflow source of truth. Extend the existing generated `Shots/<shotId>.md` hub instead of creating a second shot-note hierarchy, and add one generated `Canvas/Shot Reviews/<shotId>.canvas` per shot for spatial review. Generated shot pages remain projection-owned and tracked by `Projection Manifest.json`; user commentary remains under `Notes/`.

**Tech Stack:** TypeScript, fs-extra, Vitest, Markdown, YAML frontmatter, Obsidian wikilinks/embeds, Obsidian Bases YAML, JSON Canvas.

---

## Scope

In scope:

- Enrich each generated `Shots/<shotId>.md` into an immersive review page.
- Embed or link the shot storyboard, Step 4 image prompt, Step 5 video prompt, global execution plan, review dashboard, production board, Bases, and user note entry.
- Add shot-level frontmatter properties useful for Bases: `review_mode`, `review_canvas`, `review_note`, `has_storyboard`, `has_image_prompt`, `has_video_prompt`.
- Add per-shot JSON Canvas files under `Canvas/Shot Reviews/`.
- Add Shot Index and Project Home links to the single-shot immersive review flow.
- Add Bases support for immersive shot review fields.
- Extend `verify-obsidian` to validate generated shot review pages and per-shot review canvases.
- Add tests and docs.

Out of scope:

- No Obsidian plugin.
- No reverse sync from Obsidian notes to Step files.
- No image/video platform calls.
- No LibTV work.
- No `.obsidian/` default writes beyond the existing opt-in suggestion mode.

## Task 1: Enrich Shot Hub Pages

2. Compute per-shot links for:
   - storyboard
   - image prompt
   - video prompt
   - global execution plan
   - previous shot
   - next shot
   - `Canvas/Shot Reviews/<shotId>.canvas`
   - `Notes/Shot Reviews/<shotId>`

## Task 2: Add Per-Shot Review Canvas

1. Add `shotReviewCanvasPath(shotId: string)` in `canvas.ts`.
2. Add `renderShotReviewCanvases(sourceFiles: ObsidianSourceFile[])`.
3. For each shot, create file nodes for existing:
   - `Shots/<shotId>.md`
   - storyboard note
   - image prompt note
   - video prompt note
   - `03_Production_Board.md`
   - `Notes/README.md`
4. Add text nodes for any missing shot-stage file so the canvas remains valid without pointing to missing files.
5. Add labeled edges for `review start`, `frame`, `image prompt`, `video prompt`, `execute`, and `notes`.
6. Include these canvases in `exportObsidianVault()`.
7. Run `pnpm --filter ./apps/cli test -- obsidian-export.test.ts obsidian-verify.test.ts`.
