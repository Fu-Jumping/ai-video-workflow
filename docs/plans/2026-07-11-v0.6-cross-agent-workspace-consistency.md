# v0.6 Cross-Agent Workspace Consistency Implementation Plan

## Goal

让同一个 `ai-video-workflow` 项目目录可以被 Codex、Cursor、Claude Code、Trae、Cherry Studio、Obsidian 投影和 MCP 上下文共同读取，同时保持单一事实源和清晰写入边界。

本阶段交付：

- `AGENTS.md` 成为项目级跨智能体根入口。
- `docs/ai-workspace/` 成为项目内共享 AI 文档目录。
- 现有 IDE runtime mirror 继续保留，但入口文案统一指向 `AGENTS.md` 和 `docs/ai-workspace/`。
- Cherry Studio 先进入 adapter contract 和文档，不作为 `sync --ide` 目标。
- `verify` 和 `doctor` 能诊断共享入口缺失、共享文档缺失、平台入口冲突。

## Architecture Decisions

1. 保留 `Ide = codex | cursor | claude-code | trae`。
   - `Ide` 继续只表示可由 `sync --ide` 生成 runtime mirror 的 IDE 目标。
   - Cherry Studio 不加入 `Ide`，避免把工作目录型 Agent 和 IDE mirror 混为一类。

2. 新增 `AdapterId`。
   - 类型为 `Ide | "obsidian" | "mcp" | "cherry-studio"`。
   - 仅用于 adapter contract 和文档表达，不改变现有 `sync --ide` CLI。

3. `AGENTS.md` 是共享入口，平台目录是 runtime mirror。
   - `AGENTS.md` 只写共同规则、读取顺序和事实源边界。
   - `.codex/`、`.cursor/`、`.claude/`、`.trae/` 只写平台落位和运行提示。
   - 平台入口可引用 `AGENTS.md`，但不能定义冲突规则。

4. `docs/ai-workspace/` 是项目内共享 AI 文档层。
   - 它属于项目目录里的协作文档，不是仓库文档站 `docs/en` / `docs/zh` 的替代。
   - `sync` 采用 `writeFileIfMissing` 保护用户已有内容。
   - `verify` 通过固定 marker 发现缺失或疑似旧版入口。

5. Cherry Studio 保守接入。
   - 根据官方文档，Cherry Studio Agent 有工作目录、权限模式、autonomous mode 与 workspace `soul.md` 相关能力。
   - Cherry Studio Global Memory 仍是 Beta，且官方文档说明当前只支持 Assistant，Cherry Agent 尚不支持 global memory。
   - `@cherry/memory` MCP 需要 `MEMORY_FILE_PATH`，属于宿主或用户配置。
   - 因此本阶段只写 contract 和使用文档，不生成 `soul.md`、`user.md`、`memory/` 或私有记忆路径。

## Atomic Implementation Steps

### Step 1: Add shared workspace constants and content builders

1. Add constants for shared entry paths:
   - `AGENTS.md`
   - `docs/ai-workspace/README.md`
   - `docs/ai-workspace/BOUNDARIES.md`
   - `docs/ai-workspace/HANDOFFS.md`
   - `docs/ai-workspace/SECURITY.md`
   - `docs/ai-workspace/PLATFORM_MATRIX.md`
2. Add one reusable function in `sync.ts`, for example `ensureSharedAgentWorkspace(projectRoot)`.
3. Use `writeFileIfMissing` for all shared files.
4. Add stable marker strings:
   - `ai-video-workflow shared agent entry`
   - `docs/ai-workspace`
   - `project-step-files`
   - `runtime mirror`
   - `platform memory is not project truth`
5. Keep all generated links relative.

### Step 2: Call shared workspace setup from sync

1. In `syncProject`, call shared setup before `syncIde`.
2. Do not add a new CLI option.
3. Do not overwrite existing user `AGENTS.md` or existing shared docs.
4. Keep current platform mirror copying behavior unchanged except for entrypoint text.

### Step 3: Update platform entry text

1. Codex:
   - `.codex/README.md`, `.codex/agent-rules.md`, `.codex/repo-context.md` mention `AGENTS.md` and `docs/ai-workspace/`.
2. Cursor:
   - `.cursor/rules/ai-video-workflow.mdc` mentions `AGENTS.md` and `docs/ai-workspace/`.
3. Claude Code:
   - `CLAUDE.md` says it is a Claude Code entry and must follow `AGENTS.md`.
   - `.claude/commands/ai-video-workflow.md` points to shared docs and Step files.
4. Trae:
   - Root `AGENTS.md` is no longer Trae-only.
   - `.trae/rules/ai-video-workflow.md` contains Trae-specific runtime mirror guidance and points back to shared docs.

### Step 4: Add AdapterId and contract schema fields

1. Add `AdapterId` in `types.ts`.
2. Keep existing `Ide` type and CLI choices unchanged.
3. Extend schema with optional arrays:
   - `sharedEntryPoints`
   - `generatedSurfaces`
   - `userOwnedSurfaces`
   - `privateRuntimeSurfaces`
4. Update contract test to assert new fields are accepted and Cherry Studio fixture exists.

### Step 5: Add Cherry Studio adapter contract and docs

1. Create `cherry-studio.contract.json`.
2. Use `adapterId: "cherry-studio"`.
3. Use `syncDirection: "read-only-context"` for this stage.
4. Outputs should be documentation/context surfaces, not generated runtime files.
5. Forbidden writes must include:
   - `soul.md`
   - `user.md`
   - `USER.md`
   - `SOUL.md`
   - `memory/`
   - `MEMORY_FILE_PATH`
   - Cherry Studio global memory
   - `@cherry/memory`
   - platform caches
   - `.obsidian/`
   - absolute links
6. Add English and Chinese Cherry Studio integration docs.
7. Add contributor docs for cross-agent workspace consistency.
8. Update VitePress sidebar and index links.

### Step 6: Add shared workspace verification

1. Add issue codes:
   - `missing-shared-agent-entry`
   - `invalid-shared-agent-entry`
   - `missing-shared-agent-doc`
   - `invalid-shared-agent-doc`
   - `agent-runtime-conflict`
2. Verify `AGENTS.md` exists and contains required markers.
3. Verify every required `docs/ai-workspace/*.md` exists and contains required markers.
4. Verify platform runtime entry files mention the shared root:
   - `AGENTS.md`
   - `docs/ai-workspace/`
   - `project-step-files` or Step 1 to Step 6 facts.
5. Detect obvious conflicts:
   - platform entry says a runtime mirror is the source of truth
   - platform entry says Obsidian, MCP, or memory is project truth
   - Trae root `AGENTS.md` contains old Trae-only marker without shared markers

### Step 7: Update doctor guidance

1. Group new issue codes under `Shared Agent Workspace`.
2. Missing shared entry/docs:
   - suggest `ai-video-workflow sync --project <path> --ide <id>`.
3. Invalid existing user file:
   - suggest manually merging shared workspace markers instead of overwriting.
4. Runtime conflict:
   - suggest regenerating the platform runtime mirror and preserving user-owned files.

### Step 8: Update tests

1. `sync.test.ts`
   - assert each IDE sync writes shared workspace files.
   - assert existing `AGENTS.md`, `docs/ai-workspace/README.md`, and `CLAUDE.md` are not overwritten.
   - assert platform entries contain shared markers.
   - assert sequential sync for all IDEs keeps all runtime roots and shared docs.
2. `verify.test.ts`
   - assert synced projects pass.
   - assert deleting `AGENTS.md` reports `missing-shared-agent-entry`.
   - assert deleting a shared doc reports `missing-shared-agent-doc`.
   - assert conflicting runtime source wording reports `agent-runtime-conflict`.
3. `doctor.test.ts`
   - assert shared workspace remediation appears.
4. `agent-adapter-contract.test.ts`
   - assert Cherry Studio contract fixture exists and forbidden writes include memory/persona boundaries.

### Step 9: Update example and docs surface

1. Regenerate tracked example runtime only if tests or verify require it.
2. Update package docs links and VitePress sidebar.
3. Update pack integration README files to mention shared workspace docs.
4. Keep `.gitignore` unchanged unless tests reveal generated private files need explicit ignores.

## Verification Commands

Run after implementation:

```bash
pnpm build
pnpm test
pnpm example:verify
pnpm example:obsidian
pnpm example:obsidian:ui

```
