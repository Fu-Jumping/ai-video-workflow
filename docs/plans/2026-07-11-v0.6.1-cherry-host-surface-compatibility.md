# v0.6.1 Cherry Studio Host Surface Compatibility Plan

## Goal

Make `ai-video-workflow` projects compatible with Cherry Studio work directories that automatically create root-level `SOUL.md`, `USER.md`, and `memory/`, while keeping `AGENTS.md`, `docs/ai-workspace/`, and Step 1 to Step 6 files as the project coordination and creative source layers.

Reference model checked locally: `G:/source/南方见` uses `AGENTS.md`, `SOUL.md`, `USER.md`, `memory/`, and `docs/` as a layered multi-agent entry system. For `ai-video-workflow`, this pattern should be adapted conservatively because Cherry Studio may generate these files as host/persona/memory surfaces rather than project-owned truth.

## Architecture Decisions

1. Keep `AGENTS.md` as the cross-agent root entry for `ai-video-workflow`.
2. Keep `docs/ai-workspace/` as the shared AI documentation layer.
3. Treat `SOUL.md`, `USER.md`, `soul.md`, `user.md`, and `memory/` as Cherry Studio host/user surfaces when they appear at the project root.
4. Do not generate, overwrite, delete, or rewrite Cherry Studio host surfaces from `sync`.
5. Do not require Cherry Studio host surfaces to exist.
6. Exclude Cherry Studio host surfaces from source Markdown link checks so private/persona/memory content does not break project verification.
7. Do not add a global `.gitignore` rule yet because some projects may intentionally version these files as a collaboration protocol, like the local `南方见` reference.

## Atomic Implementation Steps

1. Add Cherry Studio host-surface language to generated shared workspace docs:
   - `AGENTS.md`
   - `docs/ai-workspace/README.md`
   - `BOUNDARIES.md`
   - `HANDOFFS.md`
   - `SECURITY.md`
   - `PLATFORM_MATRIX.md`
2. Keep `sync` write behavior unchanged for those host files:
   - no generated `SOUL.md`
   - no generated `USER.md`
   - no generated `memory/`
   - no deletion or overwrite
3. Update `verify` Markdown traversal:
   - ignore root `SOUL.md`, `USER.md`, `soul.md`, and `user.md`
   - ignore root `memory/`
   - keep scanning normal Step files and shared docs
4. Add tests:
   - `sync` preserves pre-existing Cherry host files and memory contents.
   - `verify` passes when Cherry host files contain absolute or local links that would otherwise be invalid in project docs.
   - `verify` still reports absolute links in normal project source Markdown files.
5. Update bilingual docs:
   - explain that Cherry Studio may auto-create these files.
   - explain how other agents should treat them.
   - state that versioning or ignoring them is a project/team policy, not a CLI default.
6. Regenerate or manually align the official example shared workspace baseline.
