# v0.3.6 Obsidian Vault QA and Release Hardening Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Make the Obsidian projection easier to verify as a real, user-opened vault before adding any new Obsidian concepts.

**Architecture:** Keep the existing one-way projection model. Strengthen the current `verify-obsidian` checks for optional UI suggestion files, add a repeatable official-example UI smoke script, and document the manual vault acceptance route for releases.

**Tech Stack:** TypeScript CLI, Vitest, fs-extra, VitePress documentation, generated Obsidian Markdown/Bases/Canvas/JSON config files.

---

## Scope

This release hardens the current Obsidian experience after v0.3.5. It does not introduce a plugin, reverse sync, LibTV integration, direct Obsidian app automation, agent execution, or new Step file contracts.

## Architecture Decisions

- Treat `verify-obsidian` as the machine gate for exported vault structure.
- Keep `export-obsidian --include-obsidian-ui` optional and non-authoritative.
- Validate generated suggested UI files under `.obsidian/ai-video-workflow-suggested/` more deeply than user-owned direct `.obsidian/*.json` files.
- Do not fail verification because a user has their own `.obsidian/bookmarks.json` or `.obsidian/workspace.json`.
- Add a package script for the official sample vault with optional UI suggestions, so release checks can cover both default projection and opt-in opening experience.
- Keep all checks local and file-based. Do not launch Obsidian automatically in this stage.

**Step 2: Add a JSON traversal helper**

Add a helper that recursively extracts string values from parsed JSON so the verifier can inspect Bookmarks and Workspace without depending on every Obsidian internal shape:

```ts
function collectJsonStrings(value: unknown, strings: string[] = []): string[] {
  if (typeof value === "string") {
    strings.push(value);
  } else if (Array.isArray(value)) {
    for (const item of value) {
      collectJsonStrings(item, strings);
    }
  } else if (value && typeof value === "object") {
    for (const item of Object.values(value)) {
      collectJsonStrings(item, strings);
    }
  }
  return strings;
}
```

**Step 3: Add a JSON path safety helper**

Add a helper that checks parsed JSON string values for absolute local paths and forbidden URI schemes:

```ts
function containsUnsafePathString(value: unknown): boolean {
  return collectJsonStrings(value).some((item) => !isRelativeVaultPath(item) && /\.(md|canvas|base|json)$/.test(item));
}
```

**Step 4: Strengthen suggested UI verification**

Replace the current `verifyOptionalUiConfig` body with logic that:

- Parses every direct `.obsidian/*.json` file if present.
- Parses every `.obsidian/ai-video-workflow-suggested/*.json` file if present.
- If the suggested directory exists, requires `bookmarks.json`, `workspace.json`, `core-plugins.json`, and `appearance.json`.
- Requires suggested `bookmarks.json` to include all `requiredBookmarkPaths`.
- Requires suggested `workspace.json` to include all `requiredWorkspacePaths`.
- Reports `invalid-obsidian-ui-config` for invalid JSON, missing suggested files, missing required route paths, or absolute paths in generated/suggested UI config.
- Does not require or inspect user-owned direct `.obsidian/bookmarks.json` route content beyond JSON parse and path safety.
