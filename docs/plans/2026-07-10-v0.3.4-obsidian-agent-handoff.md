# v0.3.4 Obsidian Agent Handoff Implementation Plan

## Goal

Turn the Obsidian projection into a viewing and context handoff surface for agent-driven edits. Users should inspect shots in Obsidian, then describe changes in an agent conversation while the agent edits source Step files, not generated Obsidian projection files.

## Architecture Decisions

- Keep Step 1 to Step 6 project files as the only workflow source of truth.
- Keep Obsidian one-way projection only: no reverse sync, no plugin, no generated per-shot user note bodies.
- Add generated handoff guidance as projection-owned Markdown, Bases, and Canvas navigation.
- Reuse the current `export-obsidian` and `verify-obsidian` public commands; do not add CLI flags.
- Preserve vault-relative links and JSON Canvas relative file node paths.

## Atomic Steps

1. Add shot-level handoff helpers in `dashboard.ts`.
   - Compute storyboard, Step 4 image prompt, Step 5 video prompt, and execution plan links for each shot.
   - Render a `## Agent Handoff` section in each `Shots/<shotId>.md`.
   - Include copy-ready prompt blocks for checking the shot and asking the agent to edit source Step files only.
   - Include edit boundaries: Step 3 for narrative frame, Step 4 for image/frame consistency, Step 5 for motion and camera behavior.
   - Include verification commands: `verify`, `export-obsidian`, and `verify-obsidian`.

2. Add project-level handoff page in `dashboard.ts`.
   - Generate `04_Agent_Handoff.md`.
   - Include links to Project Home, Review Dashboard, Shot Index, Production Board, Bases, and per-shot review pages.
   - Include four copy-ready prompt sections: single-shot check, Step 4 image prompt edit, Step 5 video prompt edit, and full project verification.
   - State that Obsidian is for viewing/location and Step files remain source of truth.

3. Link the handoff page from existing dashboards.
   - Add `[[04_Agent_Handoff|Agent Handoff]]` to Project Home, Review Dashboard, Shot Index, and Production Board.
   - Keep `Notes/` as optional user-authored material and do not generate `Notes/Shot Reviews/<shotId>.md`.

4. Extend `Bases/Shots.base`.
   - Add `agent_handoff` property.
   - Add `Agent Handoff` table view with `shot_order`, `file.name`, `agent_handoff`, `review_canvas`, `source_path`, and `execution_status`.
   - Add `agent_handoff` frontmatter on generated shot pages as a wikilink to the shot page section or `04_Agent_Handoff`.

5. Extend Review Map Canvas in `canvas.ts`.
   - Add a file node for `04_Agent_Handoff.md`.
   - Add edges from Project Home and Shot Index to the agent handoff node.
   - Keep all file paths vault-relative.

6. Extend verification.
   - Add issue code `invalid-obsidian-agent-handoff`.
   - Require `04_Agent_Handoff.md` markers: `Agent Handoff`, `Copy-ready Prompts`, `Source Editing Boundary`, and `Verification Commands`.
   - Require each shot page to contain `## Agent Handoff`.
   - Require `Bases/Shots.base` to include `Agent Handoff`.
   - Existing Canvas verification should catch invalid or missing Canvas node targets.

7. Extend doctor guidance.
   - Group `invalid-obsidian-agent-handoff` under Obsidian Projection.
   - Recommend rerunning `export-obsidian` and editing source Step files instead of generated projection files.

8. Add tests.
   - Export test checks `04_Agent_Handoff.md` content, shot `Agent Handoff` section, copy-ready prompt text, `Bases/Shots.base` view, and Review Map canvas node.
   - Verify test removes `04_Agent_Handoff.md` and removes a shot `Agent Handoff` marker, expecting `invalid-obsidian-agent-handoff`.
   - Doctor test checks the new remediation text.

9. Update docs.
   - README and quickstarts mention agent handoff entry.
   - Obsidian projection contributor docs explain that users inspect in Obsidian and give change requests in agent chat.
   - Reaffirm no plugin, no reverse sync, no default `.obsidian/` writes, and no LibTV in this phase.

10. Run validation.
    - `pnpm build`
    - `pnpm test`
    - `pnpm example:verify`
    - `pnpm example:obsidian`
    - `pnpm verify:v0.2`
    - `git diff --check`
    - `rg -n "\[[^\]]+\]\((https?://|file://|vscode://|[A-Za-z]:|/)" README.md README.zh-CN.md docs examples/official-mini-film`
