# v0.3.2 Obsidian Review Experience Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Improve the generated Obsidian vault so creators can review project health, shot progress, execution readiness, graph relationships, and notes from a stronger project overview.

**Architecture:** Continue from the current v0.3.1 incremental export layer without resetting or committing it first. Keep Step files as the source of truth and generate richer Obsidian projection files: enhanced frontmatter, dashboard notes, Bases views, Canvas maps, and optional opt-in `.obsidian` UI suggestions. Default export must still avoid `.obsidian/`; `--include-obsidian-ui` may generate suggested local UI files without overwriting user state.

**Tech Stack:** TypeScript, Commander, fs-extra, yaml, Vitest, VitePress, Markdown, YAML frontmatter, Obsidian Bases, JSON Canvas, optional `.obsidian` JSON files.

---

## Scope

In scope:

- Enhanced project overview, review dashboard, production board, and shot hub notes.
- Additional frontmatter properties for Bases and review queues.
- Expanded `.base` files for Review Queue, Shot Progress, Execution Readiness, and Generated File Conflicts.
- New `Canvas/Review Map.canvas`.
- More Markdown internal links so Graph, Backlinks, and Outgoing Links become useful without plugins.
- Opt-in `--include-obsidian-ui` export for suggested Bookmarks, Workspace, core plugin, and appearance files.
- Safe behavior when `.obsidian` files already exist: do not overwrite; write suggestions under `.obsidian/ai-video-workflow-suggested/` and report `skipped-user-config-existing`.

Out of scope:

- Obsidian plugin development.
- Reverse sync from Obsidian back to Step files.
- LibTV support.
- Dataview, Tasks, Kanban, Excalidraw dependencies.
- Treating Obsidian UI config format as a stable public API.

## Task 1: Enrich Review Metadata

1. Add frontmatter fields to generated workflow notes and shot hubs:
   - `review_status`
   - `execution_status`
   - `stage_group`
   - `needs_attention`
   - `shot_order` when a shot id is present
2. Derive values only from source kind, step number, shot id, and projection state.
3. Add navigation links from workflow notes to Review Dashboard, Production Board, Workflow Map, and Shot Pipeline.
4. Run `pnpm --filter ./apps/cli test -- obsidian-export.test.ts`.

## Task 4: Add Opt-In Obsidian UI Suggestions

1. Add `includeObsidianUi?: boolean` to export options.
2. Add operation status `skipped-user-config-existing`.
3. Create `ui-config.ts` to render:
   - `.obsidian/bookmarks.json`
   - `.obsidian/workspace.json`
   - `.obsidian/core-plugins.json`
   - `.obsidian/appearance.json`
   - `.obsidian/ai-video-workflow-suggested/*.json`
4. Default export must not include UI files.
5. With `--include-obsidian-ui`, write direct `.obsidian` config only when the target file does not exist.
6. If a target config exists, do not overwrite it; write the same suggested file under `.obsidian/ai-video-workflow-suggested/` and add `skipped-user-config-existing`.
7. `--dry-run --include-obsidian-ui` must report planned operations without writing files.
8. Run `pnpm --filter ./apps/cli test -- obsidian-export.test.ts cli-dist.test.ts`.
