# film-workflow

## 适用场景 / Use When

- 用户没有明确说明当前属于哪一步。
- 请求同时涉及母包、镜像、adapter、影响分析或多步骤联动。
- 已完成步骤中的局部内容被修改，需要先判断是否回链或扩散。

## 触发条件 / Trigger Conditions

- 提到“工作流”“母包”“镜像”“adapter”“初始化”“分发包”“运行入口”。
- 请求跨越多个步骤边界。
- 请求命中已完成步骤中的局部修订。

## 不适用场景 / Do Not Use When

- 当前请求已经明确落在单一步骤，且没有路由歧义。
- 当前任务只是在单一步骤里补充正文，不涉及影响分析或镜像同步。

## 输入

- 用户请求
- 当前 IDE 运行入口
- 工作流主规范与质量门槛
- 必要的项目映射文件

## 输出

- 当前应进入的步骤
- 是否需要影响分析
- 是否属于母包维护
- 是否需要同步运行镜像或 skill 包

## 核心规则

- 先判断当前请求属于母包维护还是项目执行。
- 若属于母包维护，只修改母包与运行镜像，不修改项目交付层。
- 若属于项目执行，只进入一个正确步骤，不混写多个步骤。
- 若命中已完成步骤中的局部修订，先做影响分析，再决定是否扩散修订。
- 对目录化 skill 的 IDE，skill 运行入口和文档镜像入口必须分开看待。

## 与总规范的关系 / 规则来源

- 本文件是运行打包层 skill。
- 长文规则源是同名的 `packs/official-ai-video/skills-longform/*.md`。
- 若本文件与长文规则源或工作流主规范冲突，以长文规则源和主规范为准。
